# Chapter 2 – Architecture Constraints

Documents technical, organisational, and legal constraints that limit design freedom.

---

## Document Header Template

Every document added to this chapter should start with the following header:

| Field | Value |
|-------|-------|
| **Purpose & Intent** | _What this document covers and why it exists within the Architecture Constraints chapter_ |
| **Incoming** | _Sources of constraints: regulations, organisational decisions, technology standards, or Ch. 1 quality goals_ |
| **Outgoing** | _Chapters or decisions constrained by these rules (e.g. Ch. 4 Solution Strategy, Ch. 9 Architecture Decisions)_ |
