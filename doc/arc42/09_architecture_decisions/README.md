# Chapter 9 – Architecture Decisions

Records significant Architecture Decision Records (ADRs) with context, options, and rationale.

---

## Document Header Template

Every document added to this chapter should start with the following header:

| Field | Value |
|-------|-------|
| **Purpose & Intent** | _What decision this document records and why it was significant enough to document_ |
| **Incoming** | _Ch. 2 constraints, Ch. 1 quality goals, or earlier ADRs that provided the context for this decision_ |
| **Outgoing** | _Chapters or components affected by this decision (e.g. Ch. 5 Building Blocks, Ch. 8 Concepts)_ |
